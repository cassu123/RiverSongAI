One-time setup
1. Install the systemd service:


sudo cp ~/RiverSongAI/river-song.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable river-song
sudo systemctl start river-song
2. Check it's running:


sudo systemctl status river-song
Open http://localhost:8000 — the backend now serves the frontend directly, no second terminal needed.

View live logs:


journalctl -u river-song -f
Every future update
Push your changes to GitHub as normal, then on your machine:


~/RiverSongAI/deploy.sh
That script pulls from GitHub, installs any new Python/npm deps, rebuilds the frontend, and restarts the service in one shot.

Note on the .env file — it's loaded automatically by the service but is gitignored, so your API keys never touch GitHub. You manage it manually on the machine. If you ever set this up on a second machine, you'll need to copy .env over manually.

if i own a domain how do i emplement that down the road
The standard setup is Nginx as a reverse proxy in front of your FastAPI app, with Certbot for free HTTPS. Here's the full picture:

How it works

Internet → your domain (port 443/HTTPS)
         → Nginx (reverse proxy)
         → River Song AI on localhost:8000
Your app doesn't change at all — Nginx handles the domain and SSL certificate.

When you're ready, the steps are:
1. Point your domain to your machine's IP

In your domain registrar (Namecheap, Cloudflare, etc.) add an A record:
@ → your.public.ip.address
If you're on a home network you'll also need to port-forward 80 and 443 on your router to your machine.
2. Install Nginx + Certbot


sudo apt install nginx certbot python3-certbot-nginx
3. Create an Nginx config for your domain


server {
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
The Upgrade / Connection headers are required so WebSockets (the conversation) work through the proxy.

4. Get a free SSL certificate


sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
Certbot automatically edits your Nginx config to handle HTTPS and renews the cert automatically.

5. Update your .env so CORS accepts your domain:


CORS_ORIGINS=["https://yourdomain.com"]
