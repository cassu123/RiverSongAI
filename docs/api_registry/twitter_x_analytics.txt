SERVICE: X (Twitter) API v2 — Developer Platform
WHAT IT DOES: Lets River Song pull your X/Twitter account followers, tweet impressions, likes, and retweets from the Twitter/X analytics API.
HOW TO GET IT:
  1. Go to developer.twitter.com and sign in with your X/Twitter account.
  2. Click "Sign up for Free Account" if you have not registered as a developer before.
     Fill in a use case description (e.g. "Personal analytics dashboard to track my own account metrics").
     This is reviewed instantly for the Free tier.
  3. Once approved, go to the Developer Portal -> Projects & Apps -> Create Project.
     Name the project "River Song" and select "Making a bot or automating" or "Exploring the API"
     as the use case.
  4. Create an App within the project. Copy the API Key and API Key Secret shown.
     These go in TWITTER_CLIENT_ID and TWITTER_CLIENT_SECRET.
     (In Twitter's terminology: API Key = Client ID, API Key Secret = Client Secret)
  5. In the App settings, enable OAuth 2.0 and set the redirect URI to:
       http://localhost:8000/callback/twitter
     Set the App permissions to "Read" (read-only is sufficient for analytics).
  6. Under "Keys and Tokens", generate a Bearer Token.
     This goes in TWITTER_BEARER_TOKEN. It is used for app-level read access.
  7. Also generate an Access Token and Access Token Secret (under "Authentication Tokens").
     These go in TWITTER_ACCESS_TOKEN and TWITTER_ACCESS_TOKEN_SECRET.
     These are used for user-level read access (your own account metrics).
  8. Find your numeric User ID:
     Visit tweeterid.com or call: GET https://api.twitter.com/2/users/by/username/{your_handle}
     This goes in TWITTER_USER_ID.
  Link: https://developer.twitter.com/en/docs/twitter-api
VARIABLE NAME: TWITTER_BEARER_TOKEN, TWITTER_CLIENT_ID, TWITTER_CLIENT_SECRET, TWITTER_ACCESS_TOKEN, TWITTER_ACCESS_TOKEN_SECRET, TWITTER_USER_ID
WHERE TO PUT IT: .env
NOTES:
  - X's Free tier gives 1,500 tweet reads per month. The Basic tier ($100/month) gives
    10,000 reads/month. River Song's dashboard calls are read-only and use very few reads,
    but be aware the free tier is very limited.
  - As of 2023, X/Twitter significantly restricted which analytics are available at the
    free API tier. Impressions and engagement metrics at the tweet level now require
    the Basic tier or higher. The Free tier only gives public tweet data (likes, retweets count).
  - Access tokens using OAuth 1.0a (the Twitter_ACCESS_TOKEN / SECRET pair) do not expire.
    OAuth 2.0 access tokens expire after 2 hours but can be refreshed.
    River Song can use either -- the Access Token pair is simpler since it never expires.
  - Follower count is available on the free tier via GET /2/users/{id} with user.fields=public_metrics.
  - If you primarily use X for business/brand purposes, the Professional or Business API tiers
    unlock full analytics including impressions per tweet, profile visits, and link clicks.
  - The Bearer Token gives read access to public data only. User context (your own private metrics)
    requires the OAuth 1.0a tokens or OAuth 2.0 with user context.
